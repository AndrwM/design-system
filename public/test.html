Dealer Cockpit
UI Design System & CSS Styleguide

Dealer Cockpit uses BEM CSS syntax and takes heavy cues from the ITCSS architecture strategy. "Components" in this document refer to CSS components (stylesheet for a single UI element) not Angular Components.

Each html element that has CSS styles is considered a component or a child of a component and is namespaced with a html class name to reflect the relationship. This organization strategy prevents implicit element styling by the browser and the CSS cascade. It allows each CSS component to be parent element and html-tag agnostic – components are in complete control of how they look and fill 100% width of their containing element unless otherwise defined.

Anatomy Of The System
  1. styles.scss
    Global manifest file
  2. styles/_variables.scss
    Standardized spacing, typefaces, sizing, colors, etc.
  3. styles/tools/
    SCSS mixins and functions
  4. styles/fonts/
    Brand typography
  5. styles/_base.scss
    Supplements normalize-css with project specific defaults
  6. styles/components/
    These are the meat and bones of our interface–anything visual on the page is a component.
  7. styles/utilities/
    These special classes help "bully" components in one-off situations, for example: .c-title.u-float-right. Please take special note here, if a component uses the same utility class in more than 1 or 2 instances in the app it is best to create a state class for that style variant. Also, keep in mind that most layouts can be handled by the .c-grid component.
  8. styles/vendor/
    Library-specific vendor css overrides
